Task: Estimar História de Usuário
Purpose
  • Fornecer uma estimativa de alto nível que será usada no planejamento da liberação.
Relationships
RolesPrimary Performer:
  • Programador XP
Additional Performers:
InputsMandatory:
  • None
Optional:
  • História de Usuário
    Steps
    Entender a História de Usuário

    Para fornecer uma estimativa que seja bastante acurada, os desenvolvedores precisam ter uma boa compreensão da história. Durante o planejamento da liberação, o cliente descreve cada história de usuário e apresenta os critérios para os testes de aceitação. Os desenvolvedores podem fazer perguntas até que estejam satisfeitos e compreendam os aspectos mais importantes que o cliente está solicitando. Evite a paralisia analítica; existem muitas histórias.
    Discutir Possíveis Implementações

    Às vezes, a equipe pode estimar a história com precisão. Outras vezes a história pode não se encaixar perfeitamente nas experiências anteriores, e a equipe poderá ter que explorar alternativas potenciais de design. Não decida sobre um design específico. Se várias ideias tiverem quase a mesma estimativa de tempo, escolha uma estimativa e passe para a próxima história. Evite a paralisia analítica; existem muitas histórias.

    Quando existir uma profunda incerteza, a equipe poderá pedir mais tempo para fazer pesquisas (experimento), a fim de fornecer uma estimativa mais realista.
    Fazer uma Estimativa Com Base na Experiência

    Se a equipe já tiver feito algum trabalho similar, eles simplesmente extrapolarão a partir do trabalho anterior. Para o trabalho que a equipe não estiver familiarizada, a implementação de um experimento pode fornecer informações suficientes para saber o que é possível ser feito e quanto tempo e esforço serão necessários. Uma equipe XP experiente pode estimar a maioria das histórias em poucos minutos. Evite a paralisia analítica; existem muitas histórias.
    Solicitar ao Cliente para Decompor a História

    A estimativa de uma história não poderá exceder a duração da iteração para um par de programadores. Se a estimativa for muito grande, solicite ao cliente para decompor a história. As pessoas fazem melhores estimativas para trabalhos menores. As estimativas grandes tendem a estourar o orçamento. Não é incomum decompor uma história de quatro semanas e descobrir que ela se torna quatro histórias de duas semanas.
