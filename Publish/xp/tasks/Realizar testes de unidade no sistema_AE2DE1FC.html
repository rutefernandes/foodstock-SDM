      Main Description

      Esta tarefa consiste em executar todos os testes de unidade ou aceitação para assegurar que todas as histórias de usuário previamente concluídas ainda estejam funcionando. Muitas equipes XP executam esses testes diariamente como parte da sua rotina de construção automatizada. Outras equipes executam esses testes toda vez que uma nova história é adicionada ao conjunto de histórias concluídas a fim de garantir que uma nova história não tenha causado problemas nas anteriores.

      Steps
      Executar Todos os Testes de Unidade do Sistema

      Execute testes unitários para se certificar que não foram introduzidos defeitos no código. À medida que um sistema evolui, pode tornar-se impraticável a execução de todos os testes de unidade para todas as mudanças feitas. Neste caso, você pode escolher um subconjunto dos testes de unidade para ser executado após cada mudança no código.

      Fazer Check-in do Código

      Se todos os testes de unidade do sistema passarem, faça Check-in do seu código e produza uma construção do sistema.
