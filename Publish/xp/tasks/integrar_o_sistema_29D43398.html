      Steps
      Executar Todos os Testes de Unidade do Sistema

      Enquanto você e seu parceiro estiverem trabalhando em uma tarefa, execute testes unitários para se certificar que não estejam sendo introduzidos defeitos no código. À medida que um sistema evolui, pode tornar-se impraticável a execução de todos os testes de unidade para todas as mudanças feitas. Neste caso, você pode escolher um subconjunto dos testes de unidade para ser executado após cada mudança no código. Quando uma tarefa for concluída, você achar que existe uma parte significativa de código que possa ser integrado ou for necessário por qualquer outro motivo, execute todos os testes de unidade do sistema.
      Fazer Check-in do Código

      Se todos os testes de unidade do sistema passarem, faça Check-in do seu código e produza uma construção do sistema.
