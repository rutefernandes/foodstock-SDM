Task: Definir Teste de Cliente
Purpose
  • Definir os critérios com os quais o cliente irá considerar uma história completa.
Relationships
RolesPrimary Performer:
  • Cliente XP
Additional Performers:
InputsMandatory:
  • None
Optional:
  • História de Usuário
    Steps
    Entender a História

    Visto que os testes de cliente são automatizados, o cliente deve ser muito específico sobre o que os testes irão fazer. É impossível fazer isto se o cliente não entender a história nos mínimos detalhes a partir de uma perspectiva de usuário. Escrever os testes de cliente irá forçar o cliente a percorrer todos os detalhes da história. As histórias devem ser testáveis.
    Definir Critérios de Teste

    Assim que a história estiver bem entendida e selecionada para a iteração, os testes de cliente automatizados devem ser escritos. Se a equipe cliente possuir testadores, o cliente poderá comunicar os critérios de teste ao testador. Isto pode ser feito de forma verbal ou escrita. O critério descreve os testes para o comportamento normal e excepcional do sistema.
