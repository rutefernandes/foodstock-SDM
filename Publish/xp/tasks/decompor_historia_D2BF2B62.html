Task: Decompor História
Purpose
  • Decompor as histórias de usuário em tarefas de engenharia.
Relationships
RolesPrimary Performer:
  • Programador XP
Additional Performers:
InputsMandatory:
  • None
Optional:
  • História de Usuário
    Steps
    Entender a História de Usuário

    A decomposição das histórias de usuário ocorre no planejamento da iteração. O primeiro passo no planejamento da iteração é quando o cliente percorre a história com os desenvolvedores para assegurar que eles compreendam-na. Os desenvolvedores podem fazer perguntas até que estejam satisfeitos e compreendam todos os aspectos do sistema. Os detalhes das histórias de usuário são definidos nos critérios dos testes de aceitação.
    Discutir Possíveis Implementações

    Definir as tarefas de engenharia para uma história exige uma boa compreensão de como a história será implementada. A equipe usa esse tempo para discutir possíveis alternativas ou abordagens de design.
    Identificar Tarefas de Engenharia

    Depois que a equipe tiver selecionado uma abordagem, ela se concentrará na identificação dos passos que permitam sua implementação completa durante a iteração seguinte.
