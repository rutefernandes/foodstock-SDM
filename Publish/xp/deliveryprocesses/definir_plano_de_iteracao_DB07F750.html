Task: Definir Iteração
Purpose
  • Estabelecer o que pode ser construído durante a iteração, dadas as limitações da equipe.
  • Permitir a equipe gerenciar a si própria no nível de tarefa.

Main Description
Essa tarefa consiste em que a equipe defina o que será feito e apresentado até a próxima iteração. Isso garante melhor organização para equipe, como todos os membros possuindo tarefas para serem feitas até o prazo combinado.
Steps
Precondições

Os passos para esta atividade fazem parte do planejamento de iteração XP. Para que essa atividade seja bem sucedida, as seguintes precondições devem ser cumpridas:

  • O cliente compreende muito bem as histórias de usuário.
  • O cliente definiu os critérios de aceitação das histórias.
  • Todos os membros da equipe que estarão envolvidos no desenvolvimento das histórias devem estar presentes.
O Cliente Apresenta as Histórias de Usuário

O cliente seleciona uma lista de histórias de usuário que ele gostaria que fossem incluídas na próxima iteração. As histórias normalmente vêm do plano de liberação, mas também podem ser incluídas novas histórias que não estavam planejadas originalmente. O cliente explica as histórias e os critérios dos testes de aceitação aos desenvolvedores.
Os Desenvolvedores Decompõem as Histórias

Os desenvolvedores discutem como implementar a história e decompõem-na em tarefas de engenharia. As tarefas deverão incluir todos os elementos necessários para que a história passe nos testes de aceitação do cliente.
Os Desenvolvedores Selecionam e Estimam

Os desenvolvedores selecionam todas as tarefas. Cada desenvolvedor coloca a estimativa apenas nas tarefas que tenha selecionado para si.
O Cliente Ajusta o Plano de Iteração

Se a soma das estimativas das tarefas for maior do que a soma da execução das tarefas feitas pela equipe na última iteração, o cliente deve remover algum trabalho, a fim de respeitar a velocidade da equipe.
