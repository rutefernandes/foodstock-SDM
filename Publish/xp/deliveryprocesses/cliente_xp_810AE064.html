Main Description

O papel Cliente XP tem a responsabilidade de definir qual é o produto certo a ser construído, determinar a ordem em que as funcionalidades serão construídas e certificar que o produto realmente funciona.

O Cliente XP escreve as funcionalidades do sistema, sob a forma de histórias de usuário que tenham valor de negócio. Usando o planejamento do jogo, ele escolhe a ordem em que as histórias serão construídas pela equipe de desenvolvimento. Ele também define os testes de aceitação que serão executados para provar que o sistema é confiável e faz o que é necessário.
